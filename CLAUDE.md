# DomainIQ Claude Instructions

DomainIQ is a standalone evidence-first domain research and portfolio intelligence tool.

Do not recreate the project or overwrite the repo.

Before editing:
1. Inspect git status.
2. Inspect the file tree.
3. Read only relevant files.
4. Propose the smallest safe implementation plan.

Rules:
- Do not add PDLC or Resume-JD modules to DomainIQ.
- Do not introduce new dependencies without approval.
- Do not use Tailwind, shadcn, lucide, path aliases, APIs, or env vars unless already configured.
- Preserve current behavior unless asked to change it.
- Run npm run build after changes.
- Return changed files, build result, risks, and next step.

DomainIQ should remain focused on:
- company/domain analysis
- operating model
- personas
- opportunities/triggers
- governance/risk
- Evidence Map
- pattern library
- portfolio artifact recommendations
